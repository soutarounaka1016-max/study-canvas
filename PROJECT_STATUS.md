# PROJECT_STATUS

更新日: 2026-07-20

## 現在の状態

Study Canvasは、日別の手書き計画、タスクカード、週間目標、自由ノートをまとめて使えるiPad Safari向け受験勉強管理Webアプリです。

Pull Request #27から#30までの機能は`main`へマージ済みで、利用者がiPad Safariで確認しました。手書き入力補助は、年齢確認やAPIキーを必要としない端末内OCRの検証版へ切り替えています。

## 実装済み・利用者確認済み

- 日付別手書き、月間カレンダー、自動保存
- Apple Pencil・一本指描画、二本指移動・拡大縮小
- 日別計画、週間目標、自由ノートの投げ縄選択
- 選択した手書きの移動、拡大縮小、削除
- 日付別タスクカードの追加、編集、完了、削除、自由配置
- 前日の未完了タスクを選んで今日へ繰り越す機能
- 選択した手書き範囲のプレビューと手動タスク化
- 今日やることダッシュボード
- 複数ページ自由ノートとサムネイル一覧
- 全データの統合バックアップ、項目別復元、失敗時ロールバック

## 端末内OCR検証版

### 実装済み・自動テスト対象

- OCRエンジン: Tesseract.js 7.0.0
- 日本語認識データを利用し、iPad Safariのブラウザ内で処理する構成
- 利用者が囲んだ手書き範囲だけを二値化・切り出して認識
- 選択画像は外部AIや中継サーバーへ送信しない
- 初回だけOCR本体と日本語認識データを外部配信元から取得し、ブラウザへキャッシュ
- OCR文字列から科目、勉強内容、予定時間の候補をルールで抽出
- 「数学 微積 60分」や「コンデンサー 50」のような短い入力を想定
- 読み取った文字、信頼度、警告を表示
- 候補をフォームへ入れ、利用者が確認してからタスクを保存
- 認識失敗時は手動入力へ戻り、空のタスクを自動保存しない
- 既存の手書き・タスク・週間目標・自由ノートの保存形式を変更しない

### 未完了

- iPad SafariでTesseract.jsとWebAssemblyが正常に起動するかの実機確認
- Apple Pencilで書いた日本語10件程度の認識精度測定
- 初回読み込み時間と2回目以降の速度確認
- 学校ネットワークからOCR配信元へアクセスできるかの確認
- 精度が不十分な場合の入力方法縮小（科目・時間をボタン選択、内容だけOCR）の判断

端末内OCRは印刷文字向けの技術を日本語手書きへ試す検証版です。コードが動くことと、実際の手書きで役立つ精度であることは分けて管理します。

## 手書き認識のデータ境界

- 認識対象は利用者が囲んだ手書き画像だけです。
- 端末内OCRでは画像を外部へ送信しません。
- ページ全体、他の日の計画、タスク一覧、週間目標全体、自由ノート全体を勝手に処理しません。
- 認識結果から利用者の確認なしにタスクを保存しません。
- OCRプログラムと言語データの初回取得には通信が必要です。

## 保存領域

- 日別手書き: `study-canvas:pages:v2`
- 旧手書き: `study-canvas:drawing:v1`
- 日付別タスク: `study-canvas:tasks:v1`
- 週間目標: `study-canvas:weekly:v1`
- 自由ノート: `study-canvas:free-note:v1`内のversion 2形式
- 旧Gemini接続設定: `study-canvas:ai:v1`

端末内OCRの追加で、新しいlocalStorage保存キーや既存データ移行は追加していません。
