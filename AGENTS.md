# AGENTS.md

## アプリの目的

Study Canvasは、iPadの手書きキャンバスで受験勉強の計画を作成する個人用Webアプリです。最重要価値は、今日何をするか決められることです。

## 開発ルール

- 作業前に最新mainと管理ファイルを確認する。
- mainを正式版とし、原則として作業ブランチとPull Requestを使用する。
- 承認された目的の範囲内で、作業順、変更ファイル、テスト、Pull Request数をAIが判断する。
- 修正回数に固定上限を設けない。同じ失敗を繰り返さず、原因分析、問題分割、別案への切り替えを行う。
- 無関係な機能や保存領域を変更しない。
- 実装後は`npm test`と`npm run build`を実行する。
- テスト未確認、コンフリクト状態、秘密情報を含む状態でmainへ統合しない。
- 公開コードを変えた場合は参照バージョンを更新する。
- 自動テスト、mainへの統合、公開、利用者のiPad確認を別の状態として記録する。

## iPad操作

- iPad Safariの横向きを優先する。
- Apple Pencilまたは一本指は描画と選択編集に使用する。
- 二本指はキャンバスの移動とピンチ拡大縮小に使用し、操作中は線を追加しない。
- 日別計画、週間目標、自由ノートで同じ選択・移動・拡大縮小・削除操作を提供する。
- 「今日」のランプは、表示中の日付が実際の今日である場合だけ点灯する。
- ページ一覧は月間カレンダーとして表示し、手書き済みの日を判別して開けるようにする。
- Safariの長押しメニューや意図しない拡大との競合を避ける。
- よく使う選択肢は44px以上のボタンで操作できるようにする。

## メインメニューと画面移動

- 通常URLで開いた場合はメインメニューを初期画面とする。
- ホームから、今日のキャンバス、週間目標、自由ノート、学習時間集計、ページ一覧、バックアップ・復元へ移動できるようにする。
- キャンバス画面には常設のホームボタンを置く。
- 画面の現在位置はURLハッシュで管理し、Safariの戻る・進む操作に対応する。
- 週間目標、自由ノート、ページ一覧をホームから開いた場合は、閉じるとホームへ戻す。
- ページ一覧で日付を選んだ場合は、ホームではなく選択した日のキャンバスへ移動する。
- 不明なハッシュは安全にホームへ戻す。
- 画面移動のために既存キャンバスや保存処理を複製しない。

## 保存領域

- 日別手書き: `study-canvas:pages:v2`
- 旧手書き: `study-canvas:drawing:v1`
- 日付別タスク: `study-canvas:tasks:v1`
- 週間目標: `study-canvas:weekly:v1`
- 複数ページ自由ノート: `study-canvas:free-note:v1`

保存領域は分離し、新しい値は書き込み後に読み戻して確認する。通常は既存形式や保存キーを理由なく変えない。

統合バックアップは、日別手書き、タスクとカード位置、週間目標、自由ノートを`study-canvas-full-backup` version 1として一つのJSONへ保存する。復元前に現在の全データを退避し、一つでも書き込みに失敗した場合は選択した全保存領域を変更前へ戻す。

旧形式の日別手書き専用バックアップは、日別手書きだけの復元に引き続き対応する。

## 学習時間集計

- 集計は`study-canvas:tasks:v1`から毎回計算し、同じ数値を別の保存領域へ二重保存しない。
- 週は月曜日から日曜日とする。
- 完了換算時間は、完了にしたタスクの予定時間の合計とする。
- 完了換算時間を、実際に計測した勉強時間として表示しない。
- タスクがない日や週は0として表示し、架空の達成率を作らない。
- 科目別集計は既存の科目順を維持する。

## タスク入力補助と自動認識の境界

- 利用者が明示的に囲んだ手書き範囲だけをタスク入力補助へ使用する。
- 囲んだ範囲はプレビューとして表示し、元の手書きは変更しない。
- 科目と予定時間はワンタップで選べるようにし、勉強内容は利用者が入力する。
- 利用者の確認なしにタスクを追加、変更、削除しない。
- 週間目標や自由ノートを自動で今日の予定へ変換しない。
- Tesseract.jsによる日本語自由手書きOCRは、iPad実機で全く読み取れなかったため不採用とする。
- 同じOCR方式を、実用精度の新しい根拠なしに再導入しない。
- 公開アプリからOCRプログラムや日本語認識データを読み込まない。
- GeminiとCloudflare Workerを使う旧構成は、利用条件の問題が解決するまで保留する。
- APIキーをGitHub Pagesの公開JavaScriptへ書かない。
- 外部AI接続、画像送信、費用、中継サーバーは高リスクとして明示的承認を必要とする。

## ブラウザ確認

- 主要画面の入口、保存処理、公開入口を変えた場合は、通常テストとビルドに加えてPlaywrightを実行する。
- Chromium、WebKit、iPad縦向き相当、iPad横向き相当で、起動、重大なJavaScriptエラー、主要画面移動、代表操作、再読み込み後の保存、横方向の大きなはみ出しを確認する。
- Playwrightの各テストは隔離されたブラウザコンテキストを使用し、利用者の本番localStorageを変更しない。
- 失敗時はスクリーンショット、動画、トレース、HTMLレポートをGitHub Actionsの成果物として保存する。
- mainへの反映後は、公開URLのリリース識別子と起動をGitHub Actionsで確認する。
- WebKitとiPad相当の画面サイズによる自動確認を、学校支給iPad、Apple Pencil、手書き品質、ピンチ操作の実機確認と同一視しない。

## 高リスク変更

データ移行、既存保存形式の変更、外部サービス、認証、費用、外部送信、公開方式、元に戻しにくい構造変更は、本番反映前に影響と戻し方を説明して利用者の承認を得る。

統合復元では、形式検証、復元前退避、書き込み後確認、失敗時ロールバックを省略しない。
