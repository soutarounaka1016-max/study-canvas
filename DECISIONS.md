# DECISIONS

## 2026-07-19: 新しいリポジトリで開発する

- 採用: `study-canvas`を正式な新規リポジトリとして使用する。
- 理由: 既存の科目付きToDoアプリと、手書きキャンバス中心の新アプリでは構造が大きく異なるため。
- 不採用: `codex1`全体の大規模な書き換え。既存試作を壊し、コンフリクトが増える可能性がある。

## 2026-07-19: 手書きをベクターデータとして保存する

- 採用: 1本ごとの線を座標、色、太さのデータとしてlocalStorageへ保存する。
- 理由: 画面サイズが変わっても描画し直せ、取り消し・消しゴム・将来の選択機能を実装しやすい。
- 不採用: キャンバス全体を画像だけで保存する方式。編集や将来のタスク変換が難しくなる。

## 2026-07-19: 最初のMVPは1ページに限定する

- 採用: ペン、消しゴム、履歴、自動保存を先に安定させる。
- 理由: iPadでの書きやすさとデータ保持が、後続機能すべての土台になるため。
- 保留: 日別複数ページ、週間目標、自由ノート、画像貼り付け、タスクカード、AI変換。

## 2026-07-19: アプリ内AIは入力補助に限定する

- 採用: 将来、選択した手書きを科目・内容・予定時間のタスクへ変換する。
- 不採用: AI家庭教師、週間目標からの日別タスク自動生成、承認なしの予定変更。

## 2026-07-19: GitHub Pagesは初回だけ手動で有効化する

- 採用: リポジトリ設定でPagesのソースを一度だけ「GitHub Actions」にし、その後はmain更新時に自動公開する。
- 理由: GitHub Actionsの標準トークンは、未作成のPagesサイトを初回有効化する権限を持たないため。
- 不採用: Personal Access Tokenを新規発行して保存する方式。MVP段階では秘密情報と管理負担を増やさない。

## 2026-07-19: 指す・文字入力は実装しない

- 採用: 「指す」と「文字入力」をツールバーと将来機能から外す。
- 理由: 利用者本人の確認で不要と判断し、手書きによる計画作成へ集中するため。
- 維持: 青・赤・黒の3色とペンの太さ調整は、使いやすい現行仕様を維持する。
